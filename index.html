ðŸŽ¯ Objectives â–¼

User and Admin Management:
Efficiently store, manage, and secure user information, including login credentials, personal details, and distinct user roles for both buyers and administrators.


Card Listing and Authentication:
Enable users to upload and list PokÃ©mon cards with detailed attributes (card name, set, condition, card hash) and implement a mechanism for administrators to verify the authenticity of these listings, ensuring platform reliability.


Direct Sale Transactions:
Allow users to buy and sell cards immediately at a fixed price, bypassing the auction process. This involves recording all necessary details for these direct purchases, such as the agreed-upon price, the buyer, the seller, and the card being sold.


Auction Functionality:
Allow users to initiate auctions for card listings, track starting and current prices, and manage auction durations effectively.


Bidding Management:
Record and organize bids placed on auctions, capturing the bid amount, associated bidder information, and the precise bid timestamp, while also tracking individual bidding attempts or amounts within a single bid through the user_bid entity.


Transaction Review System:
Enable users to provide feedback on completed transactions through ratings and comments, fostering trust and transparency within the platform.


Event Organization:
Provide a robust system for managing community events, including event games, descriptions, event dates, and associated promotional materials.


Card-User Association:
Accurately link each card listing and owned card to the respective user, ensuring clear ownership tracking throughout the platform.


Data Integrity and Relationship Enforcement:
Ensure the consistency, reliability, and logical connection between all entities (users, cards, listings, transactions, auctions, bids, reviews, and events) through well-defined foreign key relationships and data validation rules.


ðŸ—‚ Database Architecture â–¼
For this project, we chose to implement a relational database due to its structured nature, data integrity, and ability to efficiently handle complex relationships between entities. The platform involves various interconnected components such as users, PokÃ©mon cards, auctions, bids, and transactions, all of which require consistent and reliable linking through primary and foreign keys. A relational database provides a logical, organized way to manage these relationships while ensuring data accuracy and referential integrity. Additionally, the use of SQL allows us to perform powerful queries to support features like real-time bidding, user reputation tracking, and secure transaction logging. This makes relational databases like MySQL the ideal choice for building a scalable, secure, and data-driven marketplace system.

    Auction Functionality
  • Create an auction listing:
    • 
INSERT INTO listing (cardid, sellerid, price, listing_type, status, created_at)
VALUES (1, 1, 0.00, 'auction', 'active', NOW());

  • Initiate an auction for a listing:
    • 
INSERT INTO auction (listingid, start_time, end_time, starting_price, current_price)
VALUES (2, NOW(), DATE_ADD(NOW(), INTERVAL 7 DAY), 10.00, 10.00);

  • Retrieve active auctions with current prices:
    • 
SELECT
    a.auctionid,
    c.cardname,
    l.listingid,
    a.start_time,
    a.end_time,
    a.starting_price,
    a.current_price,
    u.username AS seller_username
FROM auction a
JOIN listing l ON a.listingid = l.listingid
JOIN card c ON l.cardid = c.cardid
JOIN user u ON l.sellerid = u.userid
WHERE l.status = 'active' AND l.listing_type = 'auction';


    Bidding Management
  • Place a new bid on an auction:
    • 
INSERT INTO bid (auctionid, bidderid, amount, bid_time)
VALUES (1, 1, 15.00, NOW());

  • Update the current price of an auction after a new bid:
    • 
UPDATE auction
SET current_price = (SELECT MAX(amount) FROM bid WHERE auctionid = 1) 
WHERE auctionid = 1;

  • Retrieve all bids for a specific auction:
    • 
SELECT
    b.bidid,
    u.username AS bidder_username,
    b.amount,
    b.bid_time
FROM bid b
JOIN user u ON b.bidderid = u.userid
WHERE b.auctionid = 1
ORDER BY b.bid_time DESC;


    Transaction Review System
  • Place a new bid on an auction:
    • 
INSERT INTO review (transactionid, reviewerid, rating, comment, created_at)
VALUES (1, 2, 5, 'Great seller, card arrived quickly and as described!', NOW());

  • Retrieve all reviews for a specific user (as a seller):
    • 
SELECT
    r.reviewid,
    t.transactionid,
    u_reviewer.username AS reviewer_username,
    r.rating,
    r.comment,
    r.created_at
FROM review r
JOIN transaction t ON r.transactionid = t.transactionid
JOIN user u_reviewer ON r.reviewerid = u_reviewer.userid
WHERE t.sellerid = 1;

  • Calculate average rating for a seller:
    • 
SELECT AVG(r.rating) AS average_rating
FROM review r
JOIN transaction t ON r.transactionid = t.transactionid
WHERE t.sellerid = 1;
