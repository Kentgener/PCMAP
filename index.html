The primary goal of this project is to develop PCMAP (PokÃ©mon Card Marketplace and Auction Platform) a web-based platform for admin-verified listing, selling, and auctioning of physical PokÃ©mon cards. This platform ensures secure transactions and transparent bidding.


Specific Objectives:
  • Allow users to register, log in, and manage their buyer profiles.
  • Allow admin to verify physical PokÃ©mon cards and store their details in the system.
  • Allow admin to create listings for verified cards with price, condition, and listing type.
  • Allow admin to set up and manage auctions, including scheduling and pricing.
  • Allow buyers to browse available listings and add items to their cart.
  • Allow buyers to place bids on cards listed in active auctions.
  • Allow buyers to purchase cards through a secure transaction process.
  • Allow buyers to leave ratings and reviews after completed transactions.
  • Allow admin to manage and post event announcements related to the platform.

    For this project, we chose to implement a relational database due to its structured nature, data integrity, and ability to efficiently handle complex relationships between entities. The platform involves various interconnected components such as users, PokÃ©mon cards, auctions, bids, and transactions, all of which require consistent and reliable linking through primary and foreign keys. A relational database provides a logical, organized way to manage these relationships while ensuring data accuracy and referential integrity. Additionally, the use of SQL allows us to perform powerful queries to support features like real-time bidding, user reputation tracking, and secure transaction logging. This makes relational databases like MySQL the ideal choice for building a scalable, secure, and data-driven marketplace system.

SELECT
    l.listing_id,
    c.cardname,
    c.cardset,
    c.condition,
    l.price,
    l.listing_type,
    u.username AS seller_username
FROM listing AS l
JOIN card AS c ON l.card_id = c.card_id
JOIN user AS u ON l.user_id = u.user_id
WHERE l.status = 'active';
